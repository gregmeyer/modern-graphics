# Example Outputs

This directory contains generated outputs from example scripts.

## Directory Structure

### `showcase/` - Curated Examples (Tracked in Git)

High-quality showcase examples suitable for README display and documentation:

- **`diagram-types/`** - One example of each diagram type (8 files)
  - `01-cycle.png` - Cycle diagram example
  - `02-comparison.png` - Comparison diagram example
  - `03-timeline.png` - Timeline diagram example
  - `04-story-slide.png` - Story slide example
  - `05-grid.png` - Grid diagram example
  - `06-flywheel.png` - Flywheel diagram example
  - `07-slide-cards.png` - Slide cards example
  - `08-slide-comparison.png` - Slide card comparison example

- **`templates/`** - Template style examples (3 files)
  - `default.png` - Default template style
  - `corporate.png` - Corporate template style
  - `tech-startup.png` - Tech startup template style

- **`attribution/`** - Attribution configuration examples (3 files)
  - `default.png` - Default attribution
  - `custom-styled.png` - Custom styled attribution
  - `with-context.png` - Attribution with context

- **`use-cases/`** - Real-world use case examples (3 files)
  - `corporate-report.png` - Corporate quarterly report example
  - `tech-pitch.png` - Tech startup pitch deck example
  - `educational-course.png` - Educational course materials example

**Regenerate showcase:** Run `python scripts/run_showcase.py` to regenerate all showcase examples. Showcase examples use prompt-based generation with default prompts (see `modern_graphics.prompt_to_diagram.DEFAULT_DIAGRAM_PROMPTS`).

### `generated/` - Temporary Outputs (Gitignored)

All example scripts output to `generated/` directory. These files are temporary and not tracked in git:

- Outputs from `all_diagram_types.py`
- Outputs from `attribution_examples.py`
- Outputs from `use_case_*.py` scripts
- Outputs from `export_options.py`
- Outputs from `batch_generation.py`
- Outputs from `temperature_effects_demo.py`
- Outputs from `story_slide_with_prompt.py`
- `prompts_used.json` files (generated by scripts for evaluation)
- Other temporary test outputs

**Note:** The `generated/` directory is completely gitignored. Files here are for local testing and development only.

## Usage

### View Showcase Examples

Showcase examples are tracked in git and can be viewed directly:

```bash
# View showcase examples
ls examples/output/showcase/diagram-types/
ls examples/output/showcase/templates/
ls examples/output/showcase/attribution/
ls examples/output/showcase/use-cases/
```

### Regenerate Showcase

To regenerate all showcase examples with the latest code:

```bash
python scripts/run_showcase.py
```

This will regenerate all showcase examples in `showcase/` directories.

### Run Example Scripts

Example scripts generate outputs to `generated/` (not tracked in git):

```bash
# Generate all diagram types
python scripts/all_diagram_types.py

# Generate attribution examples
python scripts/attribution_examples.py

# Generate use case examples
python scripts/use_case_corporate.py
```

All outputs go to `examples/output/generated/` and are automatically ignored by git.

## Showcase Criteria

Showcase examples are selected based on:

- **Visual Quality**: Clean, professional appearance
- **Clarity**: Easy to understand at a glance
- **Representative**: Shows typical use case for that diagram type
- **Data Quality**: Realistic, meaningful data (not "Test 1", "Test 2")
- **Completeness**: Shows all key features

## Maintenance

- **Showcase examples** (`showcase/`) are tracked in git and should be regenerated when:
  - Diagram generation code changes
  - Template system changes
  - Attribution system changes
  - New diagram types are added

- **Generated examples** (`generated/`) are temporary and can be deleted/recreated at any time.
